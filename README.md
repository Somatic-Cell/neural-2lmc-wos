# neural-2lmc-wos

Minimal validation code for neural-cache Walk-on-Spheres experiments.

The first milestone isolates the bias of a Neural-Caches-style estimator:

- exact analytic geometry: 3D unit ball;
- analytic harmonic solution generated by external point charges;
- GPU-batched wavefront Walk-on-Spheres;
- frozen SIREN solution cache;
- bias / variance / MSE decomposition for cached estimators.

The initial patch intentionally does **not** use neural SDFs or mesh geometry. This keeps geometry bias separated from solution-cache bias.

## Setup

Use WSL2 Ubuntu or Linux with a CUDA-enabled PyTorch installation.

```bash
python -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
# Install torch separately if you need a specific CUDA wheel.
# See https://pytorch.org/get-started/locally/
python -m pip install -e .[dev]
```

Check CUDA:

```bash
python - <<'PY'
import torch
print(torch.__version__)
print(torch.cuda.is_available())
PY
```

## Run the first validation

Train a SIREN cache against the analytic solution:

```bash
python scripts/train_siren_cache.py --config configs/ball3_siren_bias.yaml
```

Evaluate full WoS and Neural-Caches-style cached estimators:

```bash
python scripts/eval_cached_bias.py --config configs/ball3_siren_bias.yaml
```

Outputs are written to `runs/ball3_siren_bias/`:

- `siren.pt`: trained cache checkpoint;
- `cached_bias_results.json`: numeric metrics;
- `mse_vs_samples.png`: MSE curves;
- `decomp_vs_samples.png`: bias/variance decomposition.

## Current scope

This patch is deliberately limited to the following question:

> With exact geometry fixed, does a frozen neural solution cache produce a finite-depth cached estimator whose MSE converges to a nonzero bias plateau?

Later patches should add:

1. 2LMC residual correction with shared WoS prefixes;
2. tiny-cuda-nn / iNGP cache backend;
3. mesh geometry via FCPW or another exact-query backend;
4. optional neural-SDF ablation, kept separate from the main unbiased path.
